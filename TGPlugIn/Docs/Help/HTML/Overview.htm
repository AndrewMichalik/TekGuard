Why get an expensive e-mail server with a lot of features you really don't need? With EMail Interceptor, you get features you can really use.

The EMail Interceptor is a full featured Internet standards compliant, electronic mail server. It is the central repository for all electronic mail communication to and from your organization and allows your computer to communicate with other mail servers on the Internet. It acts as the post office for the Internet e-mail clients on other computers on your network. E-mail clients can communicate with the server to send and receive e-mail locally or to the world via the Internet.

Administration

Scheduled Backup - Administrators can configure EMail Interceptor to automatically backup critical system information, such as user account data, server configurations, and license data, and will store the backup file separate from the mail server. Should anything happen to EMail Interceptor or the computer it is running on, restoration is made easier with the backup file safely stored elsewhere. It is also possible to run a manual backup at any time. (V1)

Watchdog - The watchdog on EMail Interceptor ensures that SMTP, POP3, and IMAP services are always running. If, for any reason, any of the services stop, the watchdog feature will restart the service automatically. Administrators can set the time interval at which the watchdog should start downed services. By automatically monitoring these delivery protocols, EMail Interceptor is better equipped run smoothly and deliver email. (V1)

Disk Space Monitor - The Disk Space monitor will track the available hard disk space on the server computer. When the free space reaches a user specified level, the Disk Space Monitor will trigger EMail Interceptor to send notification to the server administrator. By actively tracking the disk space, EMail Interceptor protects against failure caused by a lack of memory. (V1)

Extensive Logging - 4 levels of server information logging. Separate logs can be produced for each individual service in EMail Interceptor (including POP3, SMTP, IMAP, and Control and HTTP services). As well, EMail Interceptor can produce logs showing activity by users, including the size and number of messages sent and received by each account. (Cb, V1)

ODBC (Open DataBase Connectivity) Support - EMail Interceptor supports the Windows NT/2000/XP user account database, its own database, or can connect to any ODBC compliant database (e.g. SQL®, MySQL, Access®, Oracle® databases). This simplifies setup of user accounts, mailing lists, etc., as existing lists can be imported into TekGuard: EMail Interceptor. Support for ODBC connections also allows for centralized management of user data. Instead of creating and maintaining user data in multiple applications, administrators can manage and backup the data in one place, accessing it fromEMail Interceptor with ease. This makes EMail Interceptor integration with your existing customer billing, student enrollment, HR, or other systems a snap. (Cb, V1)

Security

Spam Control- Stop Spammers from using your server with these powerful tools, and prevent unwanted email from being delivered to your system. (Cb)

Real time blacklist support:Support for various Real-time black hole lists, checking sender's IP address for each incoming SMTP connection. If the IP address is in the MAPS or ORBS database of open-relay servers identified as spammers, the incoming email is refused (see http://www.mail-abuse.org/ and http://www.ordb.org/). (V1)

SMTP authorization by IP address: Limits SMTP relay access to specified range of IP addresses. Relay access is usually allowed for users within your Local Area Network (LAN). (Cb)

Content filtering by address, subject, content or size: Allows rejection of emails from notorious spammers not known to public blacklists. Allows creation of a custom blacklist. This provides an efficient measure to fight empty "From" fields in spam emails. (Cb, V1)

The mail server can validate a sender's domain name as given in the SMTP MAIL FROM: command via a MX/DNS record lookup and reject the email message if the validation is not successful. (V1)
Kill lists based on domain, mailbox or complete address (Cb, V1)
Connection restrictions based on senders domain via the SMTP "HELO" command (Cb, V1)
Support for the refusal of anonymous connections via the SMTP "HELO" command (Cb, V1)
Connection restrictions based on sender's IP address (Cb)

"Sent email per hour" quota: Limits number of emails that one user (IP address) can send within a specific time period. Significantly reduces the risk of mail server abuse if spammer has accidentally acquired trusted user access rights. (V1)

"Concurrent connection" limit: Limits number of concurrent SMTP connections made from one IP address (one user). Protects mail server resources by limiting mail load. Prevents the use of specially designed spam programs that create multiple connections to increase the number of emails being sent to SMTP server. (V1)

Tarpitting - Administrators can configure EMail Interceptor to check for unsuccessful attempts to deliver emails to unknown users by external or local non-existing users. If the number of attempts in a session exceeds the number specified by the Admin, the IP address of the sender will be remembered for that period and no access from that IP address will be allowed within the period. (V1)

Open Relay Blocking - prevent Spammers from hijacking your email server by allowing only valid users to send mail to non-local destinations. (Cb)

POP-Authenticated Relay - Requires the client software to establish a valid POP3 login session before allowing the client to relay mail through the server via SMTP. Useful for remote users (Cb)
Restrictions on SMTP Relaying based on IP and allowed server list (Cb)
Restrictions on SMTP Relaying by requiring a valid local address in the SMTP MAIL FROM: command (Cb)
Outgoing connection can log into a POP3 mailbox on your ISP's server before SMTP messages are transferred. This is now being required by some ISPs. (POP-Authenticated Relay ) (V1)
The mail server can be set to restrict the size of incoming email messages to help prevent Email Bombing (IA)

By using authentication methods, EMail Interceptor can further protect itself from being used as a relay point. Users must identify themselves to the server and be approved to use the mail server. (Cb)

SMTP authorization by user name and password: Requires users to authenticate before sending email through SMTP.(V1)

Content Filtering - Filter incoming and outgoing emails based on a number of variables including, subject, sender and/or recipient, key words in the text, and more. Set the content filter to delete or forward filtered messages. (V1)

Multiple Domains

Multiple Service Support - EMail Interceptor will support running multiple instances of POP3, SMTP and optional IMAP servers. These servers can have their own distinct settings, mailboxes, domains and be bound to specific IP addresses. (Cb)

Multiple CPU Support - EMail Interceptor will recognize and utilize multiple processors installed on the server computer. This results in optimum performance when the server experiences heavy traffic. By taking advantage of all the available processing power, EMail Interceptor is able to handle more email faster than its competitors. (Cb)

Multiple Domains - using multiple IP addresses or not (Cb) With Multiple Domain Support, a single EMail Interceptor instance can handle several unique domains (or different addresses). If a company has several identities, all mail can be handled through one server. By reducing the administration burden inherent in running multiple email servers, all email tasks can be made more efficient, and a company can apply consistent routing and filtering rules across all of its accounts. This gives your network room to grow without affecting productivity and is especially useful for service providers with multiple clients who each want their own domain. (Cb)

Supports Alias, virtual and sub-domains (Cb)
Undeliverable mail can be forwarding to ISP, returned to sender, sent to postmaster, delivered to a catch all message box or delivered to the matching system user (Cb, V1)
Most mailboxes can be unique to a domain (user mailboxes are the exception ) (Cb, V1)

Supports Multiple Domains using three different methods:

Virtual domains where domains are all associated with one IP address (Cb)
Multi-homed domains, where domains each have their own distinct IP addresses (Cb)
A mixture of the above (Cb)
